1) как узнать работает ли celery ? - В архиве с заготовкой курсового проекта в README.rst описан процесс подготовки окружения для умного дома, там же есть строка для запуска celery, по идее если celery работает, то в консоли будет запущен "бесконечный цикл" опроса сервиса умного дома.
2) В каком виде пишутся в базу данные? каждый параметр отдельно или все вместе в одной строке? - Согласно модели данных из файла models.py, подробнее про django orm есть в лекциях 4й недели, так же в заданиях 5й недели используется работа с БД.
3) Тогда что означают: controller_name, label , value. - Из текста задания "Для сохранения настроек в базе, нужно использовать модель Settings (name, value), заготовка для которой уже есть приложенных исходниках."
controller_name - имя параметра желаемой температуры для определенного контроллера (bedroom_target_temperature и hot_water_target_temperature)

label - необязательное поле, метка или описание сохраняемого параметра

value - значение желаемой температуры контроллера

4) controller_name это имя переменной типа bedroom_light? - это имя переменной bedroom_target_temperature, hot_water_target_temperature , контроллеры bedroom_light - не требуют записи в БД.
